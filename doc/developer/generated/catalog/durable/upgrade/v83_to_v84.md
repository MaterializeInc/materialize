# catalog::durable::upgrade::v83_to_v84

Migration from catalog version 83 to 84.
Extends the Role row repair introduced in v82->v83 with a second normalization pass that prevents future drift from recurring.

**Pass 1** applies the same dangling-`-1` repair as v82->v83: for each Role with the v80-form-drift signature (a dangling `-1` plus a parsed-equal stale `+1`), it cancels the dangling retraction and retracts the stale row.

**Pass 2** normalizes every remaining live `+1` Role row whose stored bytes do not match the canonical re-serialization of its parsed value through the current proto. For cloud environments (detected by the `mz_system` cluster shape and `enable_password_auth` parameter), the `auto_provision_source` field is backfilled via the email-name heuristic before re-serializing. Rows belonging to keys that pass 1 declined to repair are skipped, as normalizing only a subset of an unrepaired key would shift the consolidation failure onto the canonical bytes. After this migration, every Role row in the shard has the byte form that future writer retractions will also produce.

`RepairStats` tracks counts of repaired, stale-retracted, normalized, and skipped rows.
