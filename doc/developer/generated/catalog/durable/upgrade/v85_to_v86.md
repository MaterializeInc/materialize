# catalog::durable::upgrade::v85_to_v86

Migration from catalog version 85 to 86.
A no-op migration: all types are JSON-compatible between v85 and v86, so no `MigrationAction`s are emitted. The only schema change at v86 is the addition of the `ClusterSystemConfiguration` and `ReplicaSystemConfiguration` collections (and their corresponding `StateUpdateKind` variants) that back scoped feature flags. Because these collections start empty, no existing data is rewritten.
