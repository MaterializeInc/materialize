# catalog::durable::upgrade::v87_to_v88

No-op migration from catalog version 87 to 88.

Every v87->v88 change is additive and confined to the append-only audit log: three new `CreateOrDropClusterReplicaReasonV1Reason` variants (`Reconfiguration`, `HydrationBurst`, `Retired`) and two new event detail types (`AlterClusterReconfigurationV1`, `ClusterHydrationBurstV1`). No existing record changes shape, so a v87-serialized record is already valid v88 and nothing is rewritten.

The `upgrade` function accepts a full v87 state snapshot and returns an empty `Vec<MigrationAction>`.
