# catalog::durable::upgrade::v88_to_v89

Migrates catalog state from version 88 to version 89.

The only structural change is adding `ReconfigurationState::status` to managed clusters. Any cluster with an in-flight reconfiguration record is rewritten to set `status` to `InProgress`. Clusters without a reconfiguration record are skipped. The remaining v88->v89 changes (new reconfiguration lifecycle transitions and hydration-burst lifecycle details in the audit log) are additive and confined to the append-only audit log, so nothing else is rewritten.

The `upgrade` function accepts a full v88 state snapshot and returns one `MigrationAction::Update` entry per cluster that carries an in-flight reconfiguration.
