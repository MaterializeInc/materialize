# mz-metrics::rusage

Collects POSIX `getrusage` statistics and publishes them as Prometheus gauges via `RuMetrics`, which is generated by the `metrics!` macro.
Tracks CPU time (user/system), maximum resident set size (converted to bytes, platform-aware for Linux vs macOS), page faults, block I/O operations, and voluntary/involuntary context switches.
Implements `MetricsUpdate` so the crate's task scheduler can periodically call `update()` at the interval configured by `MZ_METRICS_RUSAGE_REFRESH_INTERVAL`.
The struct exposes a `descs()` method returning `Vec<(String, String, Vec<String>)>` of `(name, help, label_keys)` triples for every metric it defines; this is used by `mz-metrics-catalog` to document metrics whose names are assembled at macro-expansion time and are invisible to its source scraper.
The `SOURCE` constant (`pub(crate) const SOURCE: &str = file!()`) records the source file path for the same purpose.
