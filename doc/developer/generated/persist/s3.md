# persist::s3

Implements the `Blob` trait backed by Amazon S3, with support for STS role assumption, custom endpoints, and explicit credential overrides.
Large objects are uploaded with S3 multipart upload; downloads stream individual byte-range parts concurrently into lgalloc-backed memory regions.
Exposes two `dyncfg` flags (`ENABLE_S3_LGALLOC_CC_SIZES`, `ENABLE_S3_LGALLOC_NONCC_SIZES`) to control lgalloc allocation sizes per cluster type.
`S3BlobConfig` is constructed asynchronously because SDK credential resolution may require network calls.
