# persist::s3

Implements the `Blob` trait backed by Amazon S3, with support for STS role assumption, custom endpoints, and explicit credential overrides.
Large objects are uploaded with S3 multipart upload; downloads stream individual byte-range parts concurrently.
`S3BlobConfig` is constructed asynchronously because SDK credential resolution may require network calls.
