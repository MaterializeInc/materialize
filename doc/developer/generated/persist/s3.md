# persist::s3

Implements the `Blob` trait backed by Amazon S3, with support for STS role assumption, custom endpoints, and explicit credential overrides.
Large objects are uploaded with S3 multipart upload; downloads fetch parts concurrently using the S3 `part_number` parameter on GetObject requests, reusing the original upload boundaries without requiring stored metadata.
`S3BlobConfig` is constructed asynchronously because SDK credential resolution may require network calls.
Timeout behavior is driven by a `MetricsSleep` adapter that intercepts marker `Duration` values injected into the SDK's `TimeoutConfig` and substitutes live values from `BlobKnobs`, also incrementing per-timeout-type metrics counters.
