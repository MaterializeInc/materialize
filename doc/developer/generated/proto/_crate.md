# mz-proto

Provides the `RustType` / `ProtoType` / `ProtoMapEntry` trait trio for bidirectional conversion between Rust types and prost-generated Protobuf message types, along with a unified `TryFromProtoError` error type.

## Module structure

The crate is nearly flat:

* `lib.rs` — core traits (`RustType`, `ProtoType`, `ProtoMapEntry`, `ProtoRepr`), blanket implementations for standard containers (`Vec`, `BTreeMap`, `BTreeSet`, `Option`, `Box`, `Arc`, tuples), primitive mappings, and the `protobuf_roundtrip` test helper.
* `chrono` (feature-gated) — `RustType` impls for chrono datetime types and proptest strategies for generating arbitrary date/time values.

## Key types and traits

* `RustType<Proto>` — implemented by each Rust type that has a Protobuf representation; provides `into_proto` / `from_proto`.
* `ProtoType<Rust>` — symmetric counterpart; blanket-implemented for all `P` where `Rust: RustType<P>`.
* `TryFromProtoError` — exhaustive error enum covering integer overflow, missing fields, unknown enum variants, date/regex/JSON/URL parse failures, and more.
* `IntoRustIfSome` / `TryIntoIfSome` — convenience traits for unwrapping `Option<P>` fields with a descriptive `MissingField` error.

## Dependencies and consumers

Key dependencies: `prost`, `mz-ore`, `proptest`, `regex`, `serde_json`, `globset`, `http`, `url`.
Downstream: virtually every crate that uses Protobuf serialization in Materialize implements `RustType` against types generated by this crate's build script.
