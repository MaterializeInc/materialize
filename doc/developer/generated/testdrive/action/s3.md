# testdrive::action::s3

Implements S3-related builtin commands for uploading objects to S3 (or a compatible store).
`run_upload` (`s3-file-upload`) and `run_set_presigned_url` handle general uploads and presigned URLs.
`run_upload_parquet_types` (`s3-upload-parquet-types`) builds a full Arrow schema from a column-type specification and writes a `RecordBatch`, including support for map and interval (year-month, day-time) types.
`run_upload_parquet_unsorted_map` (`s3-upload-parquet-unsorted-map`) writes a Parquet file whose map column keys are deliberately out of sorted order, for use in testing that `COPY FROM` correctly re-sorts keys on read.
For non-Parquet formats the content may be additionally compressed (bzip2, gzip, xz, zstd) before upload.
