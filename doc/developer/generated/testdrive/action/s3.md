# testdrive::action::s3

Implements S3-related builtin commands for uploading and verifying objects in S3 (or a compatible store).
`run_upload` and `run_set_presigned_url` handle general uploads and presigned URLs.
`run_upload_parquet_types` builds a full Arrow schema covering a wide range of Arrow types (integers, floats, bool, string, binary, date, timestamp, time, list, decimal, struct, uuid, interval, map) and uploads the `RecordBatch` with multiple compression variants (uncompressed, snappy, gzip, brotli, zstd, lz4).
`run_upload_parquet_unsorted_map` writes a Parquet file whose map column keys are deliberately out of sorted order, for use in testing that `COPY FROM` correctly re-sorts keys on read.
`run_verify_data` downloads all objects at a given S3 bucket and key prefix (waiting for the INCOMPLETE sentinel file to disappear) and compares their contents (CSV or Parquet rows) against the expected body.
`run_verify_keys` lists objects in a bucket prefix and retries until at least one key matches a given regex pattern.
For non-Parquet formats the content may be additionally compressed (bzip2, gzip, xz, zstd) before upload.
