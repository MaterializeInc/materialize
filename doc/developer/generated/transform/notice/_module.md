# mz-transform::notice

Defines the optimizer notice system used to surface diagnostic information to users.
`RawOptimizerNotice` is the in-pipeline enum (generated by the `raw_optimizer_notices!` macro) used by transforms; `OptimizerNotice` is the long-lived catalog representation.
`OptimizerNoticeApi` is the trait each notice type implements to provide its message, hint, and optional action.
The four concrete notice types are `EqualsNull`, `IndexAlreadyExists`, `IndexTooWideForLiteralConstraints`, and `IndexKeyEmpty`, each in its own submodule.
