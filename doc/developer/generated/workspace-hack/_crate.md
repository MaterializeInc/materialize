# workspace-hack

Auto-generated crate managed by `cargo hakari`; its sole purpose is to unify dependency feature flags across the workspace, reducing the number of distinct crate compilations.
The `lib.rs` is intentionally empty (license header only); all content lives in `Cargo.toml`, which lists pinned dependencies with the full union of features required anywhere in the workspace.
Do not edit this crate manually — regenerate with `cargo hakari generate`.
