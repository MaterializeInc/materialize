## v0.165

#### Schema branching (experimental)

* Added [`CREATE BRANCH`](/sql/create-branch),
  [`DROP BRANCH`](/sql/drop-branch),
  [`SHOW BRANCHES`](/sql/show-branches), and `SHOW BRANCH STATUS`. A branch
  forks every object in a schema (tables, materialized views, indexes,
  sources, sinks) at one coordinated point in time. Reads on a branched
  object return source data at the fork time plus branch-local writes;
  writes affect only the branch.

  Sources and sinks in a branched schema are inert by default: no
  ingestion runs and no emission happens, so branching a schema with a
  production sink cannot double-emit. Source-side `ALTER` / `DROP` /
  `RENAME` / `DROP SCHEMA` on objects a live branch references is
  blocked with a clear error naming the blocking branch.

  Gated behind a new `enable_branching` system flag, off by default.
  Behavior, syntax, and durability guarantees may change without
  notice.
