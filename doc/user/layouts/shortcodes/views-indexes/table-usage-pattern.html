Some general guidelines for usage patterns include: | Usage Pattern | General Guideline | |--------------------------------------------------------------------------------|--------------------| | View results are accessed from a single cluster only | View with an [index](/sql/create-index) | | View results are accessed across [clusters](/concepts/clusters)
(e.g., queries are served from a cluster different from the cluster that maintains the view results) | Materialized view | | Final consumer of the view is a sink or a [`SUBSCRIBE`](/sql/subscribe) operation | Materialized view | | Use of [temporal filters](/transform-data/patterns/temporal-filters/) | Materialized view |
