# sql-parser::ast

The SQL Abstract Syntax Tree module. Re-exports all AST types from `defs` and
`metadata` as a flat namespace; provides display, traversal, and transformation
infrastructure.

## Files (LOC ≈ 11,303 across this directory)

| File | What it owns |
|---|---|
| `ast.rs` | Re-export facade: `pub mod defs`, `pub mod metadata`; `pub use defs::*; pub use metadata::*` (33 LOC) |
| `metadata.rs` | `AstInfo` trait and its `Raw` implementor; `RawItemName`, `RawClusterName`, `RawNetworkPolicyName`, `RawDataType` (272 LOC) |
| `display.rs` | `AstDisplay` trait, `AstFormatter<W>`, `FormatMode` (Simple / SimpleRedacted / Stable), `WithOptionName` trait, `impl_display*` macros (348 LOC) |

## Key concepts

- **`AstInfo` trait** — the single generic parameter on all AST nodes. Associates
  resolved name types (`ItemName`, `SchemaName`, `DataType`, …). `Raw` is the
  parser's output; `Aug` (defined in `mz-sql::names`) carries catalog IDs.
- **`AstDisplay` / `AstFormatter`** — AST-specific alternative to `fmt::Display`
  that threads `FormatMode` through the tree without extra allocations.
  `FormatMode::Stable` forces all identifiers to be quoted (used for catalog
  persistence). `FormatMode::SimpleRedacted` strips literals for telemetry.
- **`Fold<Raw, Aug>`** — the mechanism `mz-sql::names::resolve` uses to rewrite
  a `Raw` AST into an `Aug` AST in one ownership-consuming pass.
- **Build-script generation** — `fold.rs`, `visit.rs`, and `visit_mut.rs`
  contain only `include!(concat!(env!("OUT_DIR"), "/…"))`. The actual trait
  methods are generated by `mz-walkabout` from the `defs/` type definitions at
  build time. Changes to node types propagate automatically to all three
  traversal traits.
- **Flat re-export** — consumers `use mz_sql_parser::ast::*` and get all
  types without knowing which sub-module they live in.

## Bubbled from defs/

- `WithOptionValue<T>` is the single shared value type for all 36 `WITH` option
  structs; redaction logic is concentrated there.
- `WithOptionName` (36 impls) is defined in `display.rs`; the per-enum impls
  live in `defs/` and carry PII-audit comments. No compile-time enforcement
  of completeness.
- The build-script boundary restricts `defs/` to pure AST types; all logic
  lives outside it.
